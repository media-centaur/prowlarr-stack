# Usenet (SABnzbd) Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add SABnzbd as a usenet download client, wired into Prowlarr exactly the way qBittorrent is, so the stack is usable with usenet once the user obtains a news-server account and Newznab indexer keys.

**Architecture:** A new `sabnzbd` service runs direct-via-ISP (not in gluetun's netns), sharing qBittorrent's two bind mounts. Prowlarr (inside gluetun's netns) reaches it via the host LAN IP on port 8085. A pre-seeded `DownloadClients` row plus a migration wire it into Prowlarr; `setup` generates the SAB API key, injects it into both SABnzbd's config and Prowlarr's DB row, and injects news-server credentials from `.env` — mirroring the existing qBittorrent password/DB-patch phase.

**Tech Stack:** Docker Compose, SQLite (Prowlarr DB), Bash (scripts, migrations, tests), Python (INI editor), SABnzbd INI config.

**Spec:** `docs/superpowers/specs/2026-07-10-usenet-sabnzbd-support-design.md`

## Reference facts (verified against the codebase)

- **`DownloadClients` schema:** `Id, Enable, Name, Implementation, Settings, ConfigContract, Priority, Categories` (no `Tags` column). `Categories` defaults to `'[]'`.
- **qBittorrent seed row** ships with placeholder `host`/`password`; `setup` patches them at install time. SAB follows the same "placeholder in seed, patched by setup" model.
- **Prowlarr → download-client reachability:** Prowlarr shares gluetun's netns. qBittorrent-direct is reached via `host = LAN_IP`. SAB is always direct, so its row's `host` is **always** `LAN_IP` (never `127.0.0.1` — no VPN branch).
- **Port 8085:** qBittorrent already publishes `8080`; SAB maps `8085:8080`.
- **`setup` structure:** Phase 4 seeds `config/` from a `files=(...)` map (setup:261-266). Phase 5 resolves the qBittorrent password (setup:286-296). Phase 6 patches the Prowlarr DB (setup:306-318). The final summary block prints per-client credentials (setup:400-430).
- **`backup`/`restore`:** `backup` tars the whole `config/` tree — SAB config captured automatically. `restore` re-runs `./setup --non-interactive` and `write_manifest` — SAB re-wired automatically. **No changes to `backup` or `restore`.**
- **`MANIFEST`:** generated by `write_manifest` in `scripts/lib/common`. Image lines auto-derive from `docker-compose.yml`; container-name lines and config-file lines are hardcoded.
- **Test harness:** `scripts/test` sources each `tests/*.test` in a subshell and runs every `test_*` function. Success = exit 0 (use `set -e` semantics; a failed assertion via `[[...]]`/`grep -q` fails the test).

#!/usr/bin/env bash
# 0003-add-sabnzbd-download-client — ensure a SABnzbd download-client row exists
# in the Prowlarr DB so usenet grabs have somewhere to go. The row ships with
# placeholder host/port/apiKey; ./setup patches them (host=LAN IP, port=8085,
# generated API key). Idempotent: no-op if a Sabnzbd row already exists or the
# DB is absent.
#
# Usage: 0003-add-sabnzbd-download-client <install_dir>
set -euo pipefail

here="$(cd "$(dirname "$0")" && pwd)"
# shellcheck source=../scripts/lib/common
source "$here/../scripts/lib/common"

root="${1:?usage: 0003-add-sabnzbd-download-client <install_dir>}"
db="$root/config/prowlarr/prowlarr.db"

if [[ ! -f "$db" ]]; then
  log_info "no prowlarr.db — skipping (fresh or non-prowlarr install)"
  exit 0
fi

# Canonical SABnzbd DownloadClient row. Settings placeholders (host/port/apiKey)
# are rewritten by scripts/patch-prowlarr-sab during ./setup. Field names come
# from Prowlarr's downloadclient schema (SabnzbdSettings); see plan Task 1.
sqlite3 "$db" <<'SQL'
INSERT INTO DownloadClients (Enable, Name, Implementation, Settings, ConfigContract, Priority, Categories)
SELECT 1, 'SABnzbd', 'Sabnzbd',
  '{"host":"sabnzbd","port":8080,"useSsl":false,"urlBase":"","apiKey":"","username":"","password":"","category":"","priority":0}',
  'SabnzbdSettings', 1, '[]'
WHERE NOT EXISTS (SELECT 1 FROM DownloadClients WHERE Implementation = 'Sabnzbd');
SQL

if [[ "$(sqlite3 "$db" "SELECT COUNT(*) FROM DownloadClients WHERE Implementation='Sabnzbd';")" -lt 1 ]]; then
  die "migration did not take effect — no Sabnzbd row present"
fi
log_ok "prowlarr.db" "SABnzbd download-client row ensured"
```

#!/usr/bin/env bash
# Idempotently rewrite the SABnzbd download-client row in a Prowlarr DB.
# Usage: patch-prowlarr-sab <db_path> <host> <port> [apikey|-]
#
# SABnzbd always runs direct (never in gluetun's netns), so <host> is the host
# LAN IP. When [apikey] is '-', it is read from stdin to avoid leaking via argv.
set -euo pipefail

here="$(cd "$(dirname "$0")" && pwd)"
# shellcheck source=lib/common
source "$here/lib/common"

db="${1:-}"
host="${2:-}"
port="${3:-}"
apikey_arg="${4:-}"

if [[ -z "$db" || -z "$host" || -z "$port" ]]; then
  die "usage: patch-prowlarr-sab <db_path> <host> <port> [apikey|-]"
fi

apikey=""
if [[ -n "$apikey_arg" ]]; then
  if [[ "$apikey_arg" == "-" ]]; then apikey=$(cat); else apikey="$apikey_arg"; fi
fi

[[ -f "$db" ]] || die "DB not found: $db"
validate_ipv4 "$host" 2>/dev/null || die "host must be a valid IPv4 address (got: $host)"
{ [[ "$port" =~ ^[0-9]+$ ]] && (( port >= 1 && port <= 65535 )); } \
  || die "port must be an integer in [1, 65535] (got: $port)"

if [[ -n "$apikey" ]]; then
  esc=$(printf '%s' "$apikey" | sed "s/'/''/g")
  printf 'UPDATE DownloadClients SET Settings = json_set(Settings, %s, %s, %s, %s, %s, %s) WHERE Implementation = %s;\n' \
    "'\$.host'" "'$host'" \
    "'\$.port'" "$port" \
    "'\$.apiKey'" "'$esc'" \
    "'Sabnzbd'" \
    | sqlite3 "$db"
else
  sqlite3 "$db" "UPDATE DownloadClients SET Settings = json_set(Settings, '\$.host', '$host', '\$.port', $port) WHERE Implementation = 'Sabnzbd';"
fi

count=$(sqlite3 "$db" "SELECT COUNT(*) FROM DownloadClients WHERE Implementation='Sabnzbd' AND json_extract(Settings, '\$.host') = '$host';")
[[ "$count" -ge 1 ]] || die "patch did not take effect — no Sabnzbd row updated"
if [[ -n "$apikey" ]]; then
  actual=$(sqlite3 "$db" "SELECT json_extract(Settings, '\$.apiKey') FROM DownloadClients WHERE Implementation='Sabnzbd' LIMIT 1;")
  [[ "$actual" == "$apikey" ]] || die "apiKey patch did not take effect"
fi
```

#!/usr/bin/env python3
"""Inject the API key and news-server credentials into a sabnzbd.ini.

Usage:
  set-sab-config <sabnzbd.ini> --api-key KEY --server-host H --server-port P \
      --server-user U --connections N

The news-server password is read from stdin (trailing newline stripped).

SABnzbd's INI has nested sections: top-level [misc], [servers], and the
double-bracketed [[server1]] subsection. configparser can't parse the double
brackets, so this edits lines with a small section-aware state machine:
  * api_key         -> set in [misc]
  * host, port,
    username,
    password,
    connections     -> set in [[server1]]
Only existing keys within the target section are rewritten (the seed ini
carries them all), so [misc] host/port are never touched by server fields.
"""
import argparse
import os
import re
import sys


def main() -> int:
    p = argparse.ArgumentParser()
    p.add_argument("ini")
    p.add_argument("--api-key", required=True)
    p.add_argument("--server-host", required=True)
    p.add_argument("--server-port", required=True)
    p.add_argument("--server-user", required=True)
    p.add_argument("--connections", required=True)
    a = p.parse_args()

    password = sys.stdin.read().rstrip("\n")

    misc = {"api_key": a.api_key}
    server = {
        "host": a.server_host,
        "port": a.server_port,
        "username": a.server_user,
        "password": password,
        "connections": a.connections,
    }

    try:
        with open(a.ini) as f:
            lines = f.readlines()
    except FileNotFoundError:
        print(f"error: {a.ini} not found", file=sys.stderr)
        return 1

    section = None
    key_re = re.compile(r"^(\s*)([A-Za-z0-9_]+)(\s*=).*$")
    out = []
    for line in lines:
        stripped = line.strip()
        if stripped.startswith("[[") and stripped.endswith("]]"):
            section = ("server", stripped.strip("[]"))
        elif stripped.startswith("[") and stripped.endswith("]"):
            section = ("misc-ish", stripped.strip("[]"))
        else:
            m = key_re.match(line)
            if m:
                key = m.group(2)
                repl = None
                if section and section[0] == "misc-ish" and section[1] == "misc" and key in misc:
                    repl = misc[key]
                elif section and section[0] == "server" and key in server:
                    repl = server[key]
                if repl is not None:
                    line = f"{m.group(1)}{key}{m.group(3)} {repl}\n"
        out.append(line)

    st = os.stat(a.ini)
    tmp = f"{a.ini}.tmp.{os.getpid()}"
    with open(tmp, "w") as f:
        f.writelines(out)
    os.chmod(tmp, st.st_mode & 0o777)
    os.replace(tmp, a.ini)
    return 0


if __name__ == "__main__":
    sys.exit(main())
```

# --- Phase 6b: SABnzbd (usenet) config + DB wiring ---
# SABnzbd always runs direct, so Prowlarr reaches it at the host LAN IP:8085.
log_phase "SABnzbd (usenet)"

# Resolve/generate the SAB API key (stable across re-runs).
sab_api_key=$(read_env_value "$ENV_FILE" SABNZBD_API_KEY)
if [[ -z "$sab_api_key" ]]; then
  sab_api_key=$(python3 -c 'import secrets; print(secrets.token_hex(16))')
  write_env_value "$ENV_FILE" SABNZBD_API_KEY "$sab_api_key"
  log_ok "SABNZBD_API_KEY" "generated and saved to .env"
else
  log_ok "SABNZBD_API_KEY" "read from .env"
fi

# News-server credentials (empty until the user has a provider account).
usenet_host=$(read_env_value "$ENV_FILE" USENET_SERVER_HOST)
usenet_port=$(read_env_value "$ENV_FILE" USENET_SERVER_PORT); [[ -z "$usenet_port" ]] && usenet_port=563
usenet_user=$(read_env_value "$ENV_FILE" USENET_SERVER_USERNAME)
usenet_conns=$(read_env_value "$ENV_FILE" USENET_SERVER_CONNECTIONS); [[ -z "$usenet_conns" ]] && usenet_conns=20
usenet_pass=$(read_env_value "$ENV_FILE" USENET_SERVER_PASSWORD)

SAB_INI="config/sabnzbd/sabnzbd.ini"
if [[ -f "$SAB_INI" ]]; then
  printf '%s' "$usenet_pass" | ./scripts/set-sab-config "$SAB_INI" \
    --api-key "$sab_api_key" \
    --server-host "$usenet_host" \
    --server-port "$usenet_port" \
    --server-user "$usenet_user" \
    --connections "$usenet_conns"
  log_ok "$SAB_INI" "API key + news-server creds written"
  if [[ -z "$usenet_host" ]]; then
    log_warn "USENET_SERVER_* is empty — SABnzbd starts but can't download until you fill .env and re-run ./setup"
  fi
else
  log_warn "$SAB_INI not present — skipping SABnzbd config injection"
fi

if [[ -f config/prowlarr/prowlarr.db ]]; then
  printf '%s' "$sab_api_key" | ./scripts/patch-prowlarr-sab config/prowlarr/prowlarr.db "$lan_ip" 8085 -
  log_ok "prowlarr.db" "SABnzbd download-client host = $lan_ip:8085, API key synced"
else
  log_warn "config/prowlarr/prowlarr.db not present — skipping SABnzbd DB patch"
fi
```
